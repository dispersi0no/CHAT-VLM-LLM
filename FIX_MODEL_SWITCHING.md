# 🔧 Исправление переключения моделей

## 📋 Проблема

При нажатии кнопки переключения моделей в Streamlit возникает ошибка, потому что:
- Контейнер `qwen-qwen3-vl` существует в **Containers** (запущен)
- Но Docker образ `vllm/vllm-openai:latest` отсутствует в **Images**
- Код пытается создать новый контейнер из несуществующего образа

## ✅ Решение 1: Загрузить Docker образ (Рекомендуется)

### Вариант A: Через Python скрипт

```bash
# Запустить скрипт загрузки
python pull_vllm_image.py
```

**Что делает:**
- Проверяет Docker
- Загружает образ `vllm/vllm-openai:latest` (~8-10 GB)
- Показывает прогресс загрузки
- Время: 5-10 минут

### Вариант B: Через BAT файл (Windows)

```bash
# Двойной клик или в CMD
pull_vllm_image.bat
```

### Вариант C: Вручную через Docker

```bash
# Прямая команда Docker
docker pull vllm/vllm-openai:latest
```

**После загрузки образа:**
- Переключение моделей будет работать
- Можно создавать новые контейнеры для других моделей

---

## ✅ Решение 2: Использовать существующие контейнеры

Если не хотите загружать образ (экономия времени/трафика):

```bash
# Запустить скрипт переключения
python fix_model_switching.py
```

**Что делает:**
- Использует уже существующие контейнеры
- Останавливает все активные модели
- Запускает выбранную модель
- Не требует загрузки образа

**Ограничение:** Работает только с уже созданными контейнерами.

## 🎯 Пошаговая инструкция

### Шаг 1: Проверка текущего состояния

```bash
# Проверить существующие контейнеры
docker ps -a

# Проверить существующие образы
docker images
```

**Ожидаемый результат:**
- В `docker ps -a` видите: `qwen-qwen3-vl-2b-instruct-vllm`
- В `docker images` НЕ видите: `vllm/vllm-openai`

### Шаг 2: Выбор решения

#### Если хотите полную функциональность (создание новых моделей):

```bash
# Загрузить образ
python pull_vllm_image.py

# Подождать 5-10 минут

# Проверить
docker images vllm/vllm-openai
```

#### Если нужно быстро переключиться между существующими моделями:

```bash
# Использовать существующие контейнеры
python fix_model_switching.py

# Выбрать модель (1-4)
```

### Шаг 3: Проверка работы

```bash
# Запустить систему
python start_system.py

# Открыть Streamlit
# http://localhost:8501

# Попробовать переключить модель
```

## 🔍 Диагностика проблем

### Проблема: "Контейнер не найден"

**Причина:** Контейнер был удален или не создан

**Решение:**
```bash
# Создать контейнер заново
python start_system.py
```

### Проблема: "Образ не найден"

**Причина:** Образ `vllm/vllm-openai:latest` не загружен

**Решение:**
```bash
# Загрузить образ
python pull_vllm_image.py
```

### Проблема: "API не отвечает"

**Причина:** Модель еще загружается в GPU

**Решение:**
```bash
# Подождать 1-2 минуты
# Проверить логи
docker logs qwen-qwen3-vl-2b-instruct-vllm

# Проверить GPU
nvidia-smi
```

### Проблема: "Недостаточно памяти GPU"

**Причина:** Несколько моделей запущены одновременно

**Решение:**
```bash
# Остановить все контейнеры
docker stop $(docker ps -q)

# Запустить только одну модель
python fix_model_switching.py
```

## 💡 Рекомендации

### Для продакшена:
1. Загрузите образ один раз: `python pull_vllm_image.py`
2. Используйте `start_system.py` для запуска
3. Переключайте модели через Streamlit UI

### Для разработки:
1. Используйте `fix_model_switching.py` для быстрого переключения
2. Держите только нужные контейнеры
3. Мониторьте GPU память: `nvidia-smi`

### Для экономии ресурсов:
1. Останавливайте неиспользуемые контейнеры
2. Используйте только одну модель одновременно
3. Очищайте старые контейнеры: `docker system prune`

## 📞 Поддержка

Если проблема не решена:

1. Проверьте логи Docker: `docker logs <container_name>`
2. Проверьте GPU: `nvidia-smi`
3. Проверьте место на диске: `df -h` (Linux) или `dir` (Windows)
4. Перезапустите Docker Desktop
5. Создайте issue в репозитории
